1. Entry

1.1. What is my program?

MC Code Copier Reloaded is a Python program that can copy reward codes in Anarchia.GG Minecraft Server (OneBlock) and let you receive your reward every time. Every 30 minutes, server is sending message with 10-characters long code (e.g. LrQ9ihPNlq). My program's role is to capture and parse it.

1.2. How does it work?

This program have to scan in-game chat to look for new codes. Hopefully, Minecraft is logging all chat messages to latest.log. Program is instantly scanning it and looking for new codes. When it detects this scheme:

Plain text
Przepisz kod <code> aby otrzymać nagrodę!

it instantly knows that new code appeared. Now, it can do many things specified in config file: Copy it to clipboard, send notification, and more.

When it detects this message, it starts scanning and looking for reward info scheme. When it detects this scheme:

Plain text
Gracz <player name> jako pierwszy przepisał kod w czasie <re-write time> i otrzymał(a) 3 Klucze AFK!

It is logging reward info and it can save it to many formats.

2. Installation

2.1. Python

Program is written in Python, so it needs Python to run. In some systems, Python is installed by default, but in others, you need to install it manually.

NOTE: This documentation only shows steps for Windows and Linux. Unfortunately (or fortunately - I hate Apple), I don't have access to any Apple PC to write here about installing Python.

First, you need to check is Python already installed on your system. To do it, open terminal (on Windows - search for PowerShell in start menu; on Linux - search for Terminal) and type following command:

PowerShell / Bash
python3 --version

If this command returns something like this (version may differ):

Plain text
Python 3.14.5

then you can go to here and continue configuring everything. If it returned some red text (on Windows) or Command not Found (on Linux), then we need to install it.

2.1.1. Windows

To install Python on Windows, type following commands:

PowerShell
winget update
winget install python3

And finish the installation. If command finished, restart it and run:

PowerShell
python3 --version

Now it should print your exact Python version.

2.1.2. Linux

To install Python on Linux, you need to use package manager for specific distro:

Bash
# Ubuntu/Debian:
sudo apt update
sudo apt install python3
# Arch Linux:
sudo pacman -Sy
sudo pacman -S python3
# Fedora/Red Hat Enterprise Linux:
sudo dnf update
sudo dnf install python3

Then, you can run this command:

Bash
python3 --version

To confirm installation success.

2.2. Pip

Pip is an external Python repository with some additional modules, that aren't included in standard package. Program is using some of them, so we need to install them right now if we want to use it.

The way we will choose dependends on your OS

2.2.1. Windows

To install packages from pip, open PowerShell and type:

PowerShell
python3 -m pip install <package name>

And after the command finishes its work, package is on your system.

2.2.2. Linux

To install packages from pip, open your terminal and type command for your distro

Bash
# Ubuntu/Debian:

sudo apt install python3-pip
# Arch Linux:
sudo pacman -Sy
sudo pacman -S python-pip
# Fedora/Red Hat Enterprise Linux:
sudo dnf update
sudo dnf install python3-pip

Then, type:

Bash
pip install <package name>

2.4. Troubleshooting

In this section I'll write about all errors that may appear in this step.

1. Config file already exists

Log
[2026-06-26 19:20:57,654] [ERROR] Config file already exists!
# Return code: 73

When you receive this message, this means, that some file exists in location specified in -config flag. To fix it, simply change/set this parameter to point to a path that doesn't exist

2. Permission error

Log
[2026-06-26 19:38:46,557] [ERROR] Can't save file in <path>: Permission Denied
# Return code: 77

When you receive this communicate, this means, that you are trying to save configuration file in path, where you don't have permission to save to. To fix this, simply change path in -config parameter to path where you are able to save files

3. Configuration

In this section I'll mention everything that you need to know when configuring this program. I'll explain all sections and settings in configuration file.

Config file is basically the most important thing: in this file you are configuring program's behavior. One wrong setting - and boom: program can't detect anything. So read whole section below with attention.

3.1. Log file

Yaml
# Add path to your latest.log file
log_file: /path/to/latest.log

This is THE MOST IMPORTANT setting in whole config file! Like said in Entry, program is reading chat messages from Minecraft's latest.log file. Here, we need to specify full path to this file.

Where can I find it?

Here is the universal (and in-game) method to do it:

  1. Start the exact Minecraft instance which you are using to play on OneBlock.
  2. In main menu, go to Options -> Resource Packs -> Open pack folder
  3. Now, file explorer should appear
  4. Go exactly one folder backwards
  5. Enter logs folder
  6. Here, find latest.log file
  7. Right click on it and go to Properties
  8. Here, find your absolute path. On Windows it will typically start with C:\Users\..., on Linux - /home/...
  9. NOTE: Make sure that the path you found is an absolute path to file, not parent folder. Copied path should ALWAYS end with latest.log

When you know your latest.log location, replace /path/to/latest.log with your exact path. Some examples:

Yaml
# Windows
log_file: C:\Users\airkeyooo\AppData\Roaming\.minecraft\logs\latest.log
# Linux
log_file: /home/airkeyooo/.minecraft/logs/latest.log

NOTE: Those are only EXAMPLES. Remember, that exact path depends on your launcher and installation method, e.g. Flatpak vs System repo (on Linux)

3.2. Read n lines

Yaml
# How many lines program should read. More lines = improved efficiency, but higher CPU and disk usage
read_lines: 2

To save your CPU and RAM, program is reading lines from the end of the latest.log file (backwards). In this setting, you need to specify, how many lines backwards program should scan.

Why not just scanning 1?

Sometimes, scanning only 1 line is enough. But there are some situations, when server post this messages in the exact moment when player does it. If it was 1, program would not see that new code have just appeared.

I did a lot of testing and I found out, that 2 is enough for 99% of situations. If you don't know what you are doing, just keep default setting here. But if program sometimes ignores code, consider changing it to 3 or even 4.

3.3. Send notifications

NOTE: Currently, this option works only on Linux, and ends with error on any other OS

Yaml
# Should program send notifications about new codes?
# WARNING: Works only on Linux and ends with error on any other OS!
send_notifications: false

This option is setting should program send system notifications about new code. To do it, it is using notify-send tool. If you are on Linux and want to use this function, change false to true.

3.4. Suggest timeout

Yaml
# Suggest when user should send code to don't look suspicious
# Give value in seconds. If you don't want to use this function, comment it or set value to 0
suggest_timeout: 5

When this setting is enabled (you can enable it by simply setting any value in this option and disable it by setting it to 0), program is adding this info:

Plain text
... Send it at: HH:MM:SS

both to console logs and notifications (if they are enabled). This HH:MM:SS is hour, minute and second when program suggeests you sending the code. Unfortunately, currently program doesn't support countdown, so you need clock with seconds to use this function.

3.5. Predict next code

Yaml
# Change shuold program predict when next code will appear.

predict_next_code: false

NOTE: Belive me, there is a reason why by default this is set to false. In its current form, this is terrible. I do not recommend using this function now, but I hope it will change in future versions.

When this option is enabled, after winning information in console program is sending this message:

Log
Next code appear time prediction: HH:MM:SS

where HH:MM:SS is hour, minute and second when next code should appear.

I think I don't need to explain this option, but maybe I'll do it: when this is set to true, detected code is copied to the clipboard

3.7. Save to CSV

Yaml
# Save results to .csv file
# If you don't want to use this function, comment line below
save_to_csv: /path/to/file.csv

NOTE: If you don't want to use this function, remove/comment this whole line

When this setting is enabled, program is saving all reward information to CSV file. Its location should be specified in this parameter.

What is CSV file?

CSV (stands for comma-separated values) is a file format, where columns are separated with , or ; character. It is basically a table.

How can I read it?

You can read it with Microsoft Excel, LibreOffice/OpenOffice Calc, Google Sheets and much more.

Saved format

This program is saving data to 5 columns:

  • Code - this column is storing the exact 10-characters long code that you had to re-write
  • Nick - winner's nick
  • Time - exact re-write time in seconds
  • Timestamp - Date and time when code appeared (in ISO 8601 format, with timezone info)
  • Me - True/False value that helps recognize, was it me

3.8. My nicknames

# Set all nicknames that are yours

Here, you can list all nicknames that are yours. Each time someone wins, program will check is winner's nickname on this list. Then, it can save this info to CSV/Database, or just display info in the console.

3.9. Scan frequency

Yaml
# How frequently (in ms) chat should be scanned.
# e.g. 200 -> messages will be scanned every 200ms
# Smaller delay may improve efficiency, but will end up with higher CPU and disk usage
scan_frequency: 300

This option controls, how often program is scanning latest.log file to find new codes. If you don't know what you should type here - keep the default value. If program isn't detecting codes/rewards info, try to decrease this value. If CPU/Disk usage is too high, try to increase this value.

3.10. MySQL/MariaDB

Yaml
# MySQL DB Access config
# If you want to use MySQL, uncomment all lines below and type your credentials
# When optional is set to false, program will finish with an error when MySQL/MariaDB server is unreachable. When it is set to true, program will only warn that it can't save data to MySQL/MariaDB, but continue its work
# If the user doesn't have password (VERY UNSAFE!), leave password field blank (nothing after ':')
# Minimal required user permissions: CREATE, INSERT
# WARNING: Requires mysql.connector module, which can be installed using: `pip install mysql-connector-python`
#mysql:
# host: localhost
# port: 3306
# user: root
# password:
# database: my_database
# optional: false

What is MySQL?

MySQL is a database server. Database is used to store data in tables (like in Excel). You can insert and grab data from it using Structural Query Language (SQL)

Do I need it?

If you don't even know what is MySQL, you definitely don't.

Why someone may want to use it?

MySQL integrates with almost everything. Program is saving all wins info, so you can e.g. make a statistics with graphs and read info from database

How to configure it?

If you want to use MySQL, uncomment all config lines. You need to create database for it and type its name here. Also, enter here your DB credentials. Remember, that program needs at least CREATE and INSERT permissions on a whole database.

4. Running program

In this section I'll show you everything you need to know when running my program

4.1. Basic running

To run program with basic options, you don't need any advanced flags. Just type command below

PowerShell / Bash

After running this command, you should see something like this:

Log

This means that program is running correctly and everything is OK. And, basically, that's all! Program is running and listening for new codes. 4.2. Flags

Program have some flags, that may change its behavior. To see them all, you can run program with -help flag:

PowerShell / Bash
python3 loader.py -help

Below, I'll list all possible flags:

4.2.1. -loglevel, -v

Possible settings: quiet, critical, error, warning, info, verbose, debug

This setting can control, how many informations program is giving to the output. info is the default level. For example, if you want to see only errors:

PowerShell / Bash
python3 loader.py -loglevel error

If you want to see all communicates, including debug ones, use level debug. If you don't want to see anything, use level quiet.

4.2.2. -config

In this option, you can specify path to the configuration file. If you keep this option blank, program will search for it in another paths. Here you can see all paths, from the least priority to the most

System-specific paths

Windows:

  • C:\Users\username\AppData\Roaming\CodeCopier\codecopier.yml
  • C:\Users\username\AppData\Roaming\CodeCopier\CodeCopier.yml
  • C:\ProgramData\codecopier\config.yml
  • C:\ProgramData\codecopier\codecopier.yml
  • C:\ProgramData\CodeCopier\config.yml
  • C:\ProgramData\CodeCopier\codecopier.yml
  • C:\ProgramData\CodeCopier\CodeCopier.yml
  • C:\Users\username\AppData\Local\codecopier\config.yml
  • C:\Users\username\AppData\Local\codecopier\codecopier.yml

Path from environment variable

Program also tries to load config file from Environment Variable CODECOPIER_CONFIG

If you want to set environment variable, here is the detailed instruction, based on your OS.

On Windows:

PowerShell
setx CODECOPIER_CONFIG "C:\Users\username\not_a_standard_file_name.yml"

Then, restart your PowerShell

On Linux:

Bash
# On Bash
echo "export CODECOPIER_CONFIG=\"/path/to/config_file.yml\"" >> ~/.bashrc
# On Zsh
echo "export CODECOPIER_CONFIG=\"/path/to/config_file.yml\"" >> ~/.zshrc

NOTE: On some Linux instances modifying .bashrc or .zshrc needs root privileges. If you get Permission Error, run the same command with sudo

-config parameter

-config parameter is the most important. Even, when some file exists in system paths or env variable, -config parameter will always win.

4.2.3. -default_config

PowerShell / Bash
python3 loader.py -default_config -config <path>

You know this option, because I've mentioned it here, but maybe I'll explain it once more: this option is used to generate default config file in path specified in -config parameter. If you have some errors during this step, please go here.

4.2.4. -runasroot

PowerShell / Bash
python3 loader.py -runasroot

WARNING!!! Please avoid using this option by running program as a normal user! Remember, that you are doing it for your own responsibility!

Normally, program doesn't allow to run as root/administrator. Add this flag to run program despite having those privileges

4.2.5. -print_format, -output_format, -of

Possible settings: default, flat, ini, json, xml

This setting controls printing format. default is the default option. It higly integrates with all -show_* options - it is changing, how informations from those options are displayed.

NOTE: If you want for this program to integrate with other programs, remember to set -loglevel quiet - -show_* options will be still displayed, but program will hide all logs, keeping only -show_* visible.

If you want to use JSON format:

PowerShell / Bash
python3 loader.py -loglevel quiet -print_format json

Alone, this option is useless, so please see all options with -show_* prefix below

4.2.6. -show_statistics

Use this option to print statistics at the end in format specified in -print_format flag. For example, if you want to print statistics in flat format, use:

PowerShell / Bash
python3 loader.py -loglevel quiet -print_format flat -show_statistics
