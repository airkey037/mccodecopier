MC Code Copier Reloaded

1. Wstęp

1.1. Czym jest mój program?

MC Code Copier Reloaded jest programem napisanym w Pythonie, którego zadaniem jest kopiowanie kodów z nagrodami na serwerze Minecraft Anarchia.GG (tryb OneBlock), pozwalając użytkownikowi wygrywanie tego konkursu za każdym razem. Co 30 minut, serwer wysyła 10-znakowe kody (np. LrQ9ihPNlq). Rolą tego programu jest wyszykiwanie takich kodów oraz odpowiednie ich przetwarzanie.

1.2. Jak on działa?

Program do działania musi skanować czat gry w poszukiwaniu informacji o nowych kodach. Na szczęście, Minecraft zapisuje wszystkie wiadomości z czatu do pliku latest.log. Program ciągle skanuje ten plik w poszukiwaniu nowych kodów. Kiedy wykryje wiadomość w tym stylu:

Tekst
Przepisz kod <kod> aby otrzymać nagrodę!

od razu wie, że przyszedł nowy kod. W tym momencie, program może zrobić wiele rzeczy (wszystko można skonfigurować w pliku konfiguracyjnym): Może skopiować go do schowka, wyświetlić powiadomienie, i wiele więcej.

Kiedy będzie wiedział, że przyszedł kod, wówczas zaczyna skanować w poszukiwaniu informacji o zwycięzcy. Gdy wykryje wiadomość w tym stylu:

Tekst
Gracz <nick gracza> jako pierwszy przepisał kod w czasie <czas przepisania> i otrzymał(a) 3 Klucze AFK!

loguje tą informację oraz może zapisać ją w wielu formatach.

1.3. Historia

Nie ma jednego powodu, dlaczego napisałem ten program: pierwszy prototyp został napisany 9 grudnia 2025 roku (oczywiście w Pythonie). I... działał, ale nie za dobrze: zdarzały się bowiem sytuacje, gdy program kopiował losowe wiadomości z czatu lub losowe informacje o graczach. Potem napisałem kolejny program, ale on też nie był perfekcyjny: był bardzo niezoptymalizowany oraz używał dwóch osobnych programów komunikujących się przez D-Bus (więc działał tylko na Linuxie).

Potem zrobiłem sobie przerwę od OneBlocka na kilka miesięcy. Ale pewnego dnia postanowiłem znowu sobie tam pograć. Znalazłem mój stary program, ale on był okropny (o czym już wspominałem). Troche mi się też nudziło, ponieważ nie miałem żadnych innych rzeczy do roboty. Więc 16 maja 2026 roku zacząłem pisać nowy, totalnie odnowiony program (stąd nazwa: MC Code Copier Reloaded).

Pierwsza w pełni działająca wersja (v1.0) została ukończona już dzień później, ale potrafiła ona jedynie czytać kody. 3 godziny później, tego samego dnia, ukończyłem wersję v1.1, która miała tylko podstawowe, ale niezwykle ważne funkcje, jak kopiowanie kodów do schowka czy wyświetlanie powiadomnień. Jeżeli byłaby to moja praca, nazwałbym to "Ukończonym". Ale kocham programować, więc skończyłem ze wsparciem dla MySQLa i CSV, możliwością przewidywania czasu pojawienia się następnego kodu i wiele więcej, ale więcej o funkcjach powiem później.

Instalacja

2.1. Python

Program jest napisany w Pythonie, więc potrzebuje Pythona do działania. Na niektóych systemach, Python jest instalowany domyślnie, ale na innych musisz go doinstalować ręcznie.

INFORMACJA: Ta dokumentacja pokazuje kroki instalacji jedynie dla Linuxa i Windowsa. Niestety (albo stety - nienawidzę Apple) nie mam dostępu do żadnego komputera od Apple'a, aby zweryfikować pisane przeze mnie kroki.

Po pierwsze, musisz sprawdzić, czy Python jest już zainstalowany na twoim systemie. Aby to zrobić otwórz terminal (na Windowsie - wyszukaj PowerShell w menu start; na Linuxie - wyszukaj Terminal) i wpisz tą komendę:

PowerShell / Bash
python3 --version

Jeżeli komenda zwróciła coś takiego:

Tekst
Python 3.14.5

wówczas możesz przejść tutaj i kontynuować instalację. Jeżeli zwróciła tekst na czerwono (na Windowsie) lub Command not Found (na Linuxie), wówczas nie mamy Pythona i musimy go zainstalować.

2.1.1. Windows

Aby zainstalować Pythona na Windowsie, wpisz w terminalu poniższą komendę:

PowerShell
winget update
winget install python3

Oraz dokończ instalację. Gdy komenda zakończy działanie, sprawdź:

PowerShell
python3 --version

Teraz powinna wyświetlić się wersja Pythona na twoim systemie.

2.1.2. Linux

Aby zainstalować Pythona na Linuxie, musisz użyć komendy odpowiedniej dla twojego menadżera pakietów:

Bash
# Ubuntu/Debian:
sudo apt update
sudo apt install python3
# Arch Linux:
sudo pacman -Sy
sudo pacman -S python3
# Fedora/Red Hat Enterprise Linux:
sudo dnf update
sudo dnf install python3

Po instalacji sprawdź:

Bash
python3 --version

aby potwiedzić, że Python został zainstalowany.

2.2. Pip

Pip jest zewnętrznym repozytorium Pythona. Zawiera ono dodatkowe moduły, które nie są dodawane do standardowych plików Pythona. Mój program ich używa, więc musimy niektóre doinstalować

Droga, w jaki sposób zainstalujemy pip oraz te programy, będzie się różnić w zależności od twojego systemu operacyjnego.

2.2.1. Windows

Aby zainstalować pakiety z pip na Windowsie, otwórz PowerShell i wpisz to polecenie

PowerShell
python3 -m pip install <nazwa pakietu>

Gdy komenda skończy działać, pakiet jest na twoim systemie.

2.2.2. Linux

Aby zainstalować pakiety z pip na Linuxie, otwórz terminal i wpisz polecenie odpowiednie dla twojej dystrybucji

Bash
# Ubuntu/Debian:
sudo apt update
sudo apt install python3-pip
# Arch Linux:
sudo pacman -Sy
sudo pacman -S python-pip
# Fedora/Red Hat Enterprise Linux:
sudo dnf update
sudo dnf install python3-pip

Aby zainstalować pakiet, wpisz:

Bash
pip install <nazwa pakietu>

2.4. Rozwiązywanie problemów

W tej sekcji napiszę o wszystkich błędach, które moga pojawić się w tym kroku

1. Plik konfiguracyjny już istnieje

Log
[2026-06-26 19:20:57,654] [ERROR] Config file already exists!
# Kod wyjścia: 73

Gdy otrzymasz tą wiadomość, to oznacza, że jakiś plik istnieje już pod ścieżką podaną we fladze -config. Aby to rozwiązać, po prostu ustaw/zmień ten parametr, aby wskazywał na jeszcze nieistniejącą ścieżkę.

2. Brak uprawnień

Log
[2026-06-26 19:38:46,557] [ERROR] Can't save file in <ścieżka>: Permission Denied
# Kod wyjścia: 77

Gdy otrzymasz ten komunikat, to oznacza, że próbujesz zapisać plik konfiguracyjny tam, gdzie nie masz uprawnień do zapisania go. Aby rozwiązać ten problem, po prostu zmień ścieżkę we fladze -config na ścieżkę, gdzie możesz zapisywać pliki.

3. Konfiguracja

W tej sekcji wspomnę wszystko co musisz wiedzieć podczas konfiguracji programu oraz wyjaśnię wszystkie sekcje i ustawienia w tym pliku

Plik konfiguracyjny jest w sumie jedną z najważniejszych rzeczy w całym programie - w tym pliku ustawiasz zachowanie całego programu. Jedno złe ustawienie może sprawić, że program przestanie wykrywać kody! Więc proszę, abyś przeczytał z uwagą sekcję poniżej

3.1. Plik logów

Yaml
# Add path to your latest.log file
log_file: /path/to/latest.log

To jest NAJWAŻNIEJSZE ustawienie w całym pliku konfiguracyjnym! Jak powiedziałem na wstępie, program odczytuje wiadomości z czatu z Minecraftowego pliku latest.log. Musimy więc wprowadzić tutaj pełną ścieżkę do tego pliku.

Gdzie mogę go znaleźć?

Oto uniwersalna metoda na znalezienie tego pliku:

  1. Uruchom dokładnie tą instancję Minecrafta, której używasz do grania na OneBlocku
  2. W menu głównym wejdż w Opcje (Options) -> Paczki zasobów (Resource Packs) -> Open pack folder (Otwórz folder z paczkami)
  3. Po kliknięciu powinien otworzyć się eksplorator plików
  4. Przejdź dokładnie jeden folder w górę
  5. Wejdź do folderu logs
  6. Tutaj, znajdź plik latest.log
  7. Kliknij prawym przyciskiem na niego i kliknij na Właściwości (Properties)
  8. Tutaj, postaraj znaleźć ścieżkę bezwzględną do pliku. Na Windowsie będzie się ona zaczynać od C:\Users\..., na Linuxie - /home/...
  9. INFORMACJA: Upewnij się, że ścieżka, którą skopiowałeś, prowadzi do pliku, a nie folderu rodzica. Ścieżka skopiowana przez ciebie powinna ZAWSZE mieć na końcu latest.log

Kiedy już znasz ścieżkę do tego pliku, zastąp /path/to/latest.log dokładną ścieżką do tego pliku. Przykłady:

Yaml
# Windows
log_file: C:\Users\airkeyooo\AppData\Roaming\.minecraft\logs\latest.log
# Linux
log_file: /home/airkeyooo/.minecraft/logs/latest.log

INFORMACJA: To są jedynie PRZYKŁADY. Pamiętaj, że ta ścieżka zależy od twojego launchera i metody jego instalacji, np. Flatpak vs Systemowe repo (na Linuxie)

3.2. Czytaj n linii wstecz

Yaml
# How many lines program should read. More lines = improved efficiency, but higher CPU and disk usage
read_lines: 2

Aby oszczędzać twój procesor i pamięć RAM, program czyta linijki od końca pliku latest.log. Ta opcja pozwoli ci ustawić, ile linii program przeczyta wstecz.

Dlaczego nie skanować tylko jednej?

Czasami 1 linia jest wystarczająca. Ale są sytuacje, gdzie serwer wyśle wiadomość o nowym kodzie dokładnie wtedy, kiedy gracz wyśle jakąś wiadomość. Jeśli czytałby tylko 1 linię, program nie zobaczy, że właśnie przyszedł nowy kod.

Przeprowadziłem dużo testów. Wyszło na to, że 2 linie są wystarczające w 99% sytuacji. Jeśli nie wiesz, co robisz, pozostaw tu domyślną opcję. Ale jeżeli zdarza się często, że program zignoruje kod, rozważ powiększenie tej liczby do 3 lub nawet 4.

3.3. Wysyłaj powiadomienia

INFORMACJA: Obecnie ta opcja zadziała tylko na Linuxie i wyrzuci błąd na każdym innym systemie operacyjnym.

Yaml
# Should program send notifications about new codes?
# WARNING: Works only on Linux and ends with error on any other OS!
send_notifications: false

Ta opcja ustawia to, czy program ma wysłać powiadomienie o nowym kodzie. Używa on do tego celu narzędzia notify-send. Jeżeli jesteś na Linuxie i chcesz używać tej funkcji, zmień false na true.

3.4. Zasugeruj opóźnienie

Yaml
# Suggest when user should send code to don't look suspicious
# Give value in seconds. If you don't want to use this function, comment it or set value to 0
suggest_timeout: 5

Gdy to ustawienie jest włączone (możesz je włączyć ustawiając tu jakąkolwiek liczbę i wyłączyć ustawiając tą opcję na 0), program dodaje tą informację:

Tekst
... Send it at: HH:MM:SS

zarówno do konsoli, jak i powiadomnień (jeśli włączone). HH:MM:SS to godzina, minuta i sekunda, o której program sugeruje wysłanie wysłać kod. Niestety obecnie progam jeszcze nie odlicza, więc potrzebujesz widzieć zegar z sekundami, aby jakkolwiek sensownie używać tej funkcji.

3.5. Przewiduj następny kod

Yaml
# Change shuold program predict when next code will appear.
# Program currently needs at least 2 codes in memory to predict next one
predict_next_code: false

NOTE: Uwierz mi: jest powód, dla którego domyślnie ta opcja jest wyłączona. W swojej obecnej formie, ta opcja jest bezużyteczna. Nie polecam używać tej funkcji w jej obecnym stanie, ale mam nadzieję, że uda mi się to zmienić w przyszłych wersjach.

Kiedy ta opcja jest włączona, po informacji o wygranej program wyśle tą wiadomość:

Log
Next code appear time prediction: HH:MM:SS

HH:MM:SS to godzina, minuta i sekunda, o której kolejny kod ma przyjść (według programu).

3.6. Kopiuj do schowka

INFORMACJA: Ta opcja wymaga modułu pyperclip. Jeśli nie wiesz, jak ją zainstalować, przejdź tutaj.

Yaml
# Should program automatically copy received code to the clipboard?
# WARNING: Requires pyperclib module, which can be installed using: `pip install pyperclip`
copy_to_clipboard: false

Myślę, że nie muszę tłumaczyć, do czego służy ta opcja, ale może to zrobię: gdy ustawisz tą opcję na true, program skopiuje wykryty kod do schowka.

3.7. Save to CSV

Yaml
# Save results to .csv file
# If you don't want to use this function, comment line below
save_to_csv: /path/to/file.csv

INFORMACJA: Jeśli nie chcesz używać tej opcji, usuń/zakomentuj całą tą opcję

Gdy ta opcja jest włączona, program zapisuje wszystkie informacje o wygranych do pliku CSV. Jego lokalizację należy ustawić w tym parametrze

Czym jest plik CSV?

CSV (skrót od comma-separated values - wartości oddzielone przecinkami) jest formatem pliku, gdzie poszczególne kolumny są oddzielone od siebie znakiem , (przecinkiem) lub ; (średnikiem). Tak właściwie to jest to format tabeli.

Jak mogę to odczytać?

Możesz odczytywać to wygodnie przez Microsoft Excel, LibreOffice/OpenOffice Calc, Google Sheets i wiele więcej.

Format zapisu

Program zapisuje dane w 5 kolumnach:

  • Code - ta kolumna przechowuje dokładny 10-znakowy kod, który musiałeś przepisać
  • Nick - nick zwycięzcy
  • Time - dokładny czas przepisania kodu w sekundach
  • Timestamp - Data i godzina pojawienia się kodu (w formacie ISO 8601 z informacją o strefie czasowej)
  • Me - Wartość True/False, która pomaga rozpoznać, czy to ja zdobyłem ten kod

3.8. Moje nicki

Yaml
# Set all nicknames that are yours
# If you don't want to set your nicknames, comment/remove whole section below
nicknames:
- Nickname1
- Nickname2

Tutaj możesz wypisać wszystkie nicki które należą do ciebie. Za każdym razem, gdy ktoś wygra, program sprawdza, czy jego nick jest na tej liście. Potem może zapisać tą informację do pliku CSV/Bazy danych, lub po prostu wyświetlić tą informację w konsoli.

3.9. Częstotliwość skanowania

Yaml
# How frequently (in ms) chat should be scanned.
# e.g. 200 -> messages will be scanned every 200ms
# Smaller delay may improve efficiency, but will end up with higher CPU and disk usage
scan_frequency: 300

Ta opcja kontroluje, jak często program ma skanować plik latest.log w poszukiwaniu nowych kodów. Jeśli nie wiesz, co tutaj ustawić - zachowaj domyślną wartość. Jeśli program nie wykrywa kodów/informacji o wygranych, spróbuj obniżyć tą wartość. Jeśli użycie procesora/dysku jest za duże, spróbuj zwiększyć tą wartość.

# MySQL DB Access config

# If the user doesn't have password (VERY UNSAFE!), leave password field blank (nothing after ':')

#mysql:
# host: localhost
# port: 3306
# user: root
# password:
# database: my_database
# optional: false

Czym jest MySQL?

MySQL jest serwerem bazodanowym. Baza danych przechowuje dane w tabelach (niczym tabela w Excelu). Możesz tu wstawiać dane i je odczytywać, używając Strukturalnego Języka Zapytań (Structural Query Language - SQL)

Czy go potrzebuję?

Jeżeli nie wiesz nawet, czym jest MySQL, zdecydowanie nie potrzebujesz.

Dlaczego ktoś chciałby go użyć?

MySQL integruje się prawie ze wszystkim. Program zapisuje do niej wszystkie informacje o wygranych, więc możesz np. robić na jej podstawie statystyki i wykresy, do których dane odczyta właśnie z bazy danych.

Jak to skonfigurować?

Jeśli chcesz użyć MySQLa, odkomentuj wszystkie linie z nim związane. Musisz utworzyć dla tego programu bazę danych i wpisać ją tutaj. Musisz również wpisać tu adres i port swojego serwera oraz login i hasło użytkownika. Pamiętaj, że program potrzebuje co najmniej uprawnień do wykonywania zapytań CREATE i INSERT wewnątrz całej tej bazy danych.
